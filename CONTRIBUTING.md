# دليل المساهمة — Contributing Guide

شكراً لاهتمامك بالمساهمة في **تصميم عينات المسوح الإحصائية**! هذا الدليل يشرح كيفية إضافة دروس جديدة، إصلاح أخطاء، أو تحسين المحتوى الحالي.

---

## كيف تساهم

### 1. الإبلاغ عن مشكلة

- افتح **Issue** جديد على GitHub
- صِف المشكلة بوضوح: خطأ رياضي، كود لا يعمل، اقتراح تحسين
- أضف تسمية مناسبة: `bug`, `enhancement`, `documentation`

### 2. إضافة درس جديد

1. **اقرأ** `LESSON_TEMPLATE.md` بالكامل
2. **أنشئ فرعاً** (`feature/lesson-X-Y-name`)
3. **أنشئ المجلد** تحت `phases/NN-phase/MM-lesson-name/` مع المجلدات الفرعية:
   - `code/` — ملفات `main.R` و `main.py`
   - `docs/` — ملف `en.md` يتبع الهيكل السداسي
   - `outputs/` — مخرجات إنتاجية مع YAML frontmatter
4. **تأكد** من مطابقة قائمة التحقق في `LESSON_TEMPLATE.md`
5. **حدّث** `ROADMAP.md` و `glossary/terms.md`
6. **أرسل** Pull Request مع شرح واضح

### 3. تحسين كود موجود

- تأكد أن التعديلات لا تكسر نقاط التأكيد (`assert`/`stopifnot`)
- شغّل ملفات الاختبار في `tests/` قبل الإرسال
- حافظ على التوافق بين R و Python

### 4. إصلاح خطأ رياضي

- وثّق المرجع الأكاديمي الذي يدعم التصحيح
- حدّث الكود والنص التعليمي معاً

---

## معايير الكود

| القاعدة | التفصيل |
|---------|---------|
| **اللغة** | الكود بالإنجليزية بالكامل |
| **NA/NaN** | كل كود يتعامل مع القيم المفقودة |
| **التحقق** | `stopifnot()` في R، `assert` في Python |
| **الدقة** | تطابق حتى 4 خانات عشرية مع المكتبات الإنتاجية |
| **التعليقات** | مختصرة وشارحة، لا إسهاب |

---

## معايير النص التعليمي

- **العربية** هي لغة الشرح الأساسية
- **المصطلح الإنجليزي** يُذكر عند أول استخدام بين قوسين
- **LaTeX** للمعادلات الرياضية
- **الهيكل السداسي** إلزامي في كل درس
- **بدون إيموجي** في النصوص التعليمية

---

## تسمية الفروع

```
feature/lesson-X-Y-description   # درس جديد
fix/lesson-X-Y-bug-description   # إصلاح خطأ
docs/improve-glossary             # تحسين توثيق
```

---

## عملية المراجعة

1. يُراجَع كل Pull Request بواسطة مساهم واحد على الأقل
2. يجب أن تمرر جميع اختبارات `tests/` بنجاح
3. يجب مطابقة قائمة التحقق في `LESSON_TEMPLATE.md`
4. يُدمج في الفرع الرئيسي بعد الموافقة
